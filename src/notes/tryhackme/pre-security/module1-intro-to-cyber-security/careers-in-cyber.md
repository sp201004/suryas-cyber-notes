## Objective
Understand the major career paths in Cyber Security and the responsibilities, skills, and progression of each role.

## Cyber Security Career Map
```
Cyber Security
│
├── 🔵 Defensive Security (Blue Team)
│     ├── Security Analyst
│     ├── Security Engineer
│     ├── Incident Responder
│     ├── Threat Hunter
│     └── Malware Analyst
│
├── 🔴 Offensive Security (Red Team)
│     ├── Penetration Tester
│     ├── Red Team Operator
│     ├── Web Pentester
│     └── Network Pentester
│
└── 🟣 Purple Team
      └── Combines Red + Blue Teams
```

## Introduction
Cybersecurity offers multiple career paths, whether you enjoy attacking systems (Offensive Security) or protecting them (Defensive Security).

**Why Choose Cyber Security?**
- **High Demand** — Over **3.5 million** unfilled jobs worldwide.
- **High Salaries** — Competitive pay, even for entry-level roles.
- **Continuous Learning** — Technology and threats evolve constantly.

## Security Analyst (Blue Team)
**Role:** Monitor systems, investigate alerts, and respond to security incidents — the **digital defenders** of an organization.

**Responsibilities:** monitor security events · investigate suspicious activity · analyze alerts · respond to incidents · work with security teams · improve organizational defense.

**Skills Needed:** analytical thinking · curiosity · attention to detail · log analysis · networking basics.

**Career Progression**
```
Security Analyst → Incident Responder / Threat Hunter / Malware Analyst / SOC Lead
```

## Security Engineer
**Role:** **Design, build, and maintain** an organization's security infrastructure — creating systems that prevent attacks before they happen.

**Responsibilities:** design security architecture · maintain security systems · configure security tools · assess risks · keep defenses updated · write security documentation.

**Example:** maintaining an `Intrusion Detection System (IDS)`.

**Skills Needed:** Linux · Networking · Firewalls · IDS/IPS · Cloud Security · Scripting.

**Career Progression**
```
Security Engineer → Application Security / Cloud Security / Infrastructure Security / Security Architect
```

## Penetration Tester (Ethical Hacker)
**Role:** Legally attempts to hack systems to identify vulnerabilities before malicious attackers do.

**Goal:** Find weaknesses → Exploit safely → Report findings → Help improve security.

**Typical Workflow**
```
Reconnaissance → Enumeration → Find Vulnerabilities → Exploitation → Privilege Escalation → Report Writing
```

> **Engagement:** the agreed-upon, authorized process of penetration testing between the tester and the organization. Always requires written permission and defined scope.

**Skills Needed:** Linux · Networking · Web Security · Active Directory · Scripting · Report Writing.

## Blue Team vs Red Team
| Blue Team | Red Team |
|---|---|
| Protect systems | Attack systems ethically |
| Monitor alerts | Find vulnerabilities |
| Incident Response | Exploitation |
| SIEM | Burp Suite |
| Firewalls | Metasploit |
| IDS/IPS | Nmap |

**Blue Team:** Splunk · Microsoft Sentinel · Elastic (ELK) · Wireshark · Suricata · Snort · CrowdStrike

**Red Team:** Nmap · Burp Suite · Metasploit · Gobuster · FFUF · Hydra · SQLmap

## Interview Notes
- **Security Analyst?** Monitors events, investigates alerts, responds to incidents, and helps protect systems.
- **Security Engineer?** Designs, implements, and maintains security infrastructure (firewalls, IDS/IPS, secure architecture).
- **What is an IDS?** An Intrusion Detection System monitors activity for suspicious behavior and generates alerts.
- **Penetration Testing?** A legal, authorized simulation of cyberattacks to discover and validate vulnerabilities.
- **Engagement?** The formal agreement defining scope, rules, timeline, and authorization for a pentest.

## Common Mistakes
| Mistake | Correct Approach |
|---|---|
| Thinking cybersecurity only means hacking | It includes both offensive (Red) and defensive (Blue) roles |
| Assuming pentesters can attack any system | Testing must be authorized and within scope |
| Believing engineers only configure firewalls | They design and maintain the whole security infrastructure |
| Ignoring soft skills | Communication, documentation, teamwork, and analysis matter |

## 30-Second Revision
- Cybersecurity has **3.5M+** unfilled jobs globally.
- **Security Analyst** → Monitor, investigate, and respond to incidents.
- **Security Engineer** → Design and maintain security infrastructure (e.g., IDS).
- **Penetration Tester** → Ethically hack systems under an authorized **engagement**.
- **Blue Team** protects; **Red Team** attacks ethically.
- Strong foundations: **Networking → Linux → OS → Programming → Web Security → Cloud**.

> **Core Lesson:** Whether you enjoy **defending systems (Blue Team)**, **building secure infrastructure**, or **ethically testing defenses (Red Team)**, every role contributes to keeping organizations secure.
