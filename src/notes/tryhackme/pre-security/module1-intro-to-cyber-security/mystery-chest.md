## Security Analyst
**Mission:** Protect an organization's systems by continuously monitoring, detecting, investigating, and responding to cyber threats.

**Core Responsibilities:** monitor security alerts · analyze logs · investigate incidents · identify attack patterns · improve security posture · recommend improvements · work with engineering/IT teams · create security reports.

### Daily Workflow
```
Receive Alert → Investigate Logs → Find Root Cause → Contain Attack → Recover Systems → Write Report
```

### Required Skills
**Technical:** Networking · Linux · Windows · TCP/IP · DNS · HTTP/HTTPS · Firewalls · SIEM · IDS/IPS · Incident Response · Log Analysis

**Soft Skills:** Curiosity · Critical Thinking · Patience · Communication · Documentation · Teamwork

### Common Tools
| Category | Tools |
|---|---|
| SIEM | Splunk, Microsoft Sentinel, ELK Stack, IBM QRadar, Google Chronicle |
| Network Monitoring | Wireshark, Zeek, tcpdump |
| IDS / IPS | Snort, Suricata |
| Endpoint Security | CrowdStrike, Microsoft Defender, SentinelOne |

### Career Progression
```
Security Analyst → SOC Analyst → Incident Responder → Threat Hunter → Security Engineer → SOC Lead → Security Architect
```

### Suggested TryHackMe Learning Path
```
Pre Security → SOC Level 1 → Jr Penetration Tester
```

## Skills Roadmap for a Security Analyst
```
Computer Basics → Networking → Linux → Windows → Web Fundamentals → Security Fundamentals → Logs → SIEM → Incident Response → Threat Hunting
```

## Important Terms
| Term | Meaning |
|---|---|
| Security Analyst | Monitors and investigates security events to protect systems |
| SIEM | Security Information and Event Management — collects, correlates, and analyzes logs from multiple sources |
| Alert | Notification generated when suspicious activity is detected |
| Incident | A confirmed security event affecting confidentiality, integrity, or availability |
| Log Analysis | Reviewing system, application, and network logs to identify malicious activity |
| Threat Hunting | Proactively searching for hidden threats that bypassed traditional controls |

## Security Analyst vs Penetration Tester
| Security Analyst | Penetration Tester |
|---|---|
| Blue Team | Red Team |
| Defends systems | Tests systems by attacking ethically |
| Uses SIEM and logs | Uses tools like Nmap and Burp Suite |

## 30-Second Revision
