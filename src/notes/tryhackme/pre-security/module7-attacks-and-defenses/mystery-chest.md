## 1. MEMORY TRICKS

### The CIA Triad
```text
C → Can I SEE the data?     → Unauthorized access       → Confidentiality
I → Is the data CORRECT?    → Unauthorized modification → Integrity
A → Can I USE the data?     → Service availability      → Availability
```

### Defensive Security (PDMAR)
```text
P → Prevent
D → Detect
M → Mitigate
A → Analyze
R → Respond
```

### Defense in Depth Layers
```text
Threat Anticipation   → Prepare Early
Attack Awareness      → Know Attacker Methods
Risk Prioritization   → Protect Critical Assets
Continuous Adaptation → Keep Improving
Defense in Depth      → Multiple Security Layers
```

## 2. INTERVIEW QUESTIONS (OFFENSIVE SECURITY)

| Question | Answer |
|---|---|
| **What is CIA?** | Confidentiality, Integrity, Availability. |
| **What is Offensive Security?** | Testing systems proactively to identify vulnerabilities before attackers can exploit them. |
| **Difference between Red Team and Penetration Testing?** | Pentest is a focused assessment. Red Team is a full attacker simulation. |
| **What is Exploitation?** | Using a vulnerability to gain unauthorized access or demonstrate impact. |
| **What is a Dictionary Attack?** | A password guessing attack using a predefined list of common passwords. |
| **What is Hydra?** | A fast network login cracker used for password testing against multiple protocols. |
| **Why is Permission Important?** | Without authorization, penetration testing becomes illegal. |
| **What is Scope?** | Defines authorized systems and actions during a security assessment. |
| **What is Enumeration?** | Collecting information about the target before exploitation. |

## 3. INTERVIEW QUESTIONS (DEFENSIVE SECURITY)

| Question | Answer |
|---|---|
| **What is Defensive Security?** | Protecting systems by preventing, detecting, and responding to attacks. |
| **Who is the Blue Team?** | Security professionals responsible for protecting infrastructure and responding to attacks. |
| **Why is Visibility important?** | Because defenders cannot protect systems they cannot see or monitor. |
| **Difference between Prevention and Detection?** | Prevention stops attacks before they happen. Detection finds attacks after they begin. |
| **What is Mitigation?** | Reducing damage during or after an attack. |
| **What is Defense in Depth?** | Using multiple security layers so that if one control fails, another still protects the system. |
| **What is Risk Prioritization?** | Protecting the most valuable assets first. |
| **What is Threat Anticipation?** | Thinking ahead about how attackers may compromise systems before they attack. |
